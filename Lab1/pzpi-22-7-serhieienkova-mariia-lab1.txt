1.	BUSINESS REQUIREMENTS (БІЗНЕС ВИМОГИ)

1.1	Background (Передумови)



Сучасний світ стикається з демографічними змінами, які ведуть до збільшення частки літніх людей у суспільстві. Ця категорія населення потребує спеціального догляду, постійного моніторингу здоров’я та вчасного реагування на зміни у стані організму. Сім'ї, які піклуються про своїх літніх родичів, нерідко відчувають труднощі через відсутність ефективних інструментів для дистанційного спостереження за здоров’ям. У свою чергу, медичні працівники потребують засобів, які спрощують обробку даних і забезпечують оперативну реакцію на критичні зміни.
Програмна система для догляду за літніми людьми створюється для вирішення цієї проблеми. Вона дозволяє дистанційно відслідковувати ключові життєві показники (пульс, температуру тіла, артеріальний тиск), автоматично інформувати родичів або лікарів у разі критичних змін, а також надавати рекомендації щодо подальших дій.
Основна мета розробки — забезпечити безпеку та комфорт літніх людей, знизити емоційне та фізичне навантаження на родичів і доглядальників, а також покращити ефективність медичного нагляду.








1.2	Business Opportunity (Бізнес можливості)



Ринок технологій, орієнтованих на покращення догляду за літніми людьми, демонструє стабільне зростання. Зростаюча частка літнього населення у світі разом із поширенням IoT-технологій та цифрових рішень для охорони здоров’я створює попит на інтегровані, зручні у використанні, надійні системи для моніторингу стану здоров’я. Однак, більшість існуючих рішень на ринку мають обмежений функціонал, складний для розуміння інтерфейс, відсутність багатомовності або недостатньо інтегровані з IoT-пристроями.
Пропонована система вирізняється своєю унікальною бізнес-логікою, яка спрямована на надання комплексного рішення для моніторингу здоров’я літніх людей. Завдяки поєднанню IoT-пристроїв, аналітичних алгоритмів та розробки мобільного застосунку, вона здатна забезпечити як базовий контроль, так і критичне сповіщення про небезпечні зміни у стані здоров’я.
Існуючі аналоги системи:
-	CarePredict [1] – система для моніторингу активності літніх людей.
Вона використовує спеціальні датчики для відслідковування рухів і поведінки, але не забезпечує повного моніторингу життєвих показників (таких як пульс чи тиск);
-	GrandCare [2] – платформа для дистанційного моніторингу, яка
дозволяє	відслідковувати	стан	здоров’я	літніх	людей	через	IoT-пристрої. Водночас вона складна в налаштуванні та використанні.
Модель монетизації передбачає включати:
-	продаж IoT-пристроїв з базовим функціоналом;
-	передплату на розширені функції аналізу даних та підтримку;
-	інтеграцію	з	медичними	організаціями	для	спрощення	процесу обслуговування пацієнтів.
 
2	VISION OF THE SOLUTION (КОНЦЕПЦІЯ РІШЕННЯ)
2.1	Vision Statement (Окреслення концепції)



Система для догляду за літніми людьми спрямована на покращення якості життя цієї категорії населення шляхом впровадження сучасних технологій моніторингу здоров'я. Вона призначена для дистанційного спостереження за життєвими показниками, автоматичного інформування про критичні ситуації та надання рекомендацій щодо дій.
Метою системи є створення безпечного середовища для літніх людей, а також підтримка їх родичів і медичних працівників через інтуїтивно зрозумілий інтерфейс та інтеграцію з IoT-пристроями.
Система стане платформою, яка забезпечить повний спектр функцій для моніторингу здоров’я літніх людей, включаючи збір даних у реальному часі, аналітику та автоматичні сповіщення. Це дозволить не лише зменшити ризики, пов’язані з раптовими змінами стану здоров’я, але й покращити комунікацію між пацієнтами, їхніми близькими та медичними працівниками.



2.2	Major Features (Головна функціональність)



Система забезпечує наступну функціональність:
-	MF-1 Моніторинг життєвих показників здоров'я — система дозволяє відслідковувати показники, такі як пульс, температура тіла, артеріальний тиск. Дані збираються автоматично за допомогою інтегрованих IoT-пристроїв;
 
-	MF-2 Аналітика та рекомендації — система обробляє дані за
допомогою аналітичних алгоритмів для надання можливих ризиків. Надання рекомендацій користувачам щодо покращення стану здоров'я, система може порекомендувати додаткові обстеження;
-	MF-3 Автоматичні сповіщення — у разі виявлення критичних змін у стані здоров'я користувача (наприклад, підвищення артеріального тиску до
небезпечного рівня або аномального серцевого ритму), система автоматично інформує медичних працівників, родичів або доглядальників. Це забезпечує своєчасну реакцію та допомогу у випадках надзвичайних ситуацій;
-	MF-4 Нагадування про прийом ліків і заплановані медичні огляди, що допомагає уникнути пропусків. Така функція допомагає уникнути пропусків у лікуванні, що може бути критичним для людей похилого віку;
-	MF-5 Історія даних та звітність — система зберігає історію змін життєвих показників для аналізу довгострокових тенденцій. Генерує звіти, які можуть бути використані для консультацій із медичними фахівцями. Це також дозволяє медичному персоналу краще оцінювати ефективність лікування.
 
3. SCOPE AND LIMITATIONS (РАМКИ ТА ОБМЕЖЕННЯ ПРОЄКТУ)
3.1	Scope of Initial Release (Рамки первинного випуску)


Програмна система для догляду за літніми людьми передбачає створення комплексного рішення для дистанційного моніторингу життєвих показників таких, як пульс, температура тіла, артеріальний тиск, з використанням IoT- пристроїв.
Метою першої версії системи є забезпечення базової функціональності для моніторингу стану здоров’я, сповіщення про аномалії та управління даними. Вона буде доступна через веб-платформу і мобільний додаток, з можливістю доступу для лікарів, родичів та доглядальників. Нижче наведено функціональні можливості, які будуть включені у перший реліз.
Основні функціональні можливості першого випуску:
1.	Функціонал серверної частини:
-	FR-SRV-01: реалізація RESTful API для взаємодії клієнтської частини
з сервером і IoT-пристроями;
-	FR-SRV-02: аналіз показників здоров’я — реалізація алгоритмів
виявлення критичних змін (різких коливань пульсу, тиску);
-	FR-SRV-03: відправлення сповіщень родичам, лікарям або
доглядальникам при виявленні критичних змін у стані здоров’я користувача;
-	FR-SRV-04: надання доступу кільком ролям (родич, лікар,
доглядальник) до профілю літньої людини;
-	FR-SRV-05: обробка запитів на управління користувачами, даними про 
стан здоров’я та сповіщеннями;
-	FR-SRV-06: реєстрація, авторизація та управління профілями
користувачів;
-	FR-SRV-07: автоматична обробка даних з IoT-пристроїв та
формування рекомендацій або критичних сповіщень;
-	FR-SRV-08: імпорт та експорт даних для збереження історії медичних
показників;
-	FR-SRV-09: робота з базою даних — обробка запитів на збереження, 
оновлення, видалення та отримання даних.
2.	Функціонал IoT-пристроїв:
-	FR-IOT-01: вимірювання основних показників здоров’я (пульс,
температура тіла, артеріальний тиск) за допомогою простих датчиків;
-	FR-IOT-02: підключення пристроїв до сервера через Wi-Fi;
-	FR-IOT-03: передача зібраних даних на сервер для аналізу в режимі
реального часу;
-	FR-IOT-04: автоматичне оновлення показників для подальшого
відображення в інтерфейсі.
3.	Функціонал веб-інтерфейсу:
-	FR-WEB-01: авторизація та автентифікація користувачів (родичів,
лікарів, доглядальників);
-	FR-WEB-02: перегляд інформації про літню людину: стан здоров’я,
історія показників;
-	FR-WEB-03: додавання/редагування профілів користувачів, зокрема
літніх людей і пов’язаних з ними осіб;
-	FR-WEB-04: налаштування прав доступу для членів родини пацієнтом;
-	FR-WEB-05: перегляд та фільтрація медичних показників за періодами
часу;
-	FR-WEB-06: перегляд журналу сповіщень про критичні стани;
-	FR-WEB-07: генерація аналітичних звітів щодо змін стану здоров’я.
4.	Функціонал мобільного застосунку:
-	FR-MOB-01: авторизація та автентифікація користувачів;
-	FR-MOB-02: відображення останніх медичних показників у зручному
форматі;
-	FR-MOB-03: отримання push-сповіщень про критичні зміни стану
здоров’я;
-	FR-MOB-04: доступ до профілю літньої людини та пов’язаної
інформації;
-	FR-MOB-05: можливість швидкого зв’язку з лікарем або родичем
через вбудовану кнопку дій;
-	FR-MOB-06: перегляд історії вимірювань;
-	FR-MOB-07: налаштування сповіщень (що саме отримувати).


3.2	Scope of Subsequent Releases (Рамки наступних випусків)


У наступних версіях системи планується розширення функціональності для підвищення точності аналізу, зручності використання та адаптивності до нових медичних потреб. Особлива увага приділятиметься інтеграції алгоритмів штучного інтелекту, розширеному адмініструванню, підтримці додаткових IoT-пристроїв і підвищенню безпеки.
Основні напрямки розвитку включають:
1.	Інтелектуальний аналіз та застосування ШІ
-	впровадження алгоритмів машинного навчання для прогнозування
можливих відхилень у стані здоров’я;
-	автоматичне формування рекомендацій на основі попередніх медичних 
даних;
-	аналіз ефективності лікування, оцінка змін стану пацієнта в динаміці;
2.	Розширені функції адміністрування
-	персоналізовані налаштування сповіщень та подій;
-	доступ до розширених журналів активності та історії змін медичних
даних.
3.	Інтеграція нових IoT-пристроїв
-	підтримка пристроїв для вимірювання рівня кисню в крові, глюкози,
ЕКГ, фізичної активності тощо;
-	стандартизоване підключення та синхронізація з комерційними
медичними пристроями.
4.	Підвищення безпеки даних
-	впровадження двофакторної автентифікації;
-	шифрування всіх медичних даних під час зберігання та передачі;
-	аудит доступу та логування дій користувачів.


3.3	Limitations and Exclusions (Обмеження та виключення)


Попри готовність рішення до базового моніторингу та сповіщень,
першочерговий випуск системи має низку технічних і організаційних обмежень, а також виключених із релізу функціональних можливостей.
Неочевидні обмеження:
1.	Обмежена автономність IoT-пристроїв. Пристрої вимірювання працюють лише за наявності стабільного живлення та інтернет-з’єднання; у разі його втрати дані втрачаються до відновлення зв’язку.
2.	Моно-пацієнтський режим сенсорів. Кожен IoT-пристрій прив’язується до одного профілю літньої людини — мультипацієнтське використання не підтримується.
3.	Відсутність офлайн-режиму для клієнтських застосунків. Інтерфейси (веб та мобільний) не працюють без інтернет-підключення.
Виключений із початкової версії функціонал:
1.	Відео- та аудіо-консультації з лікарем. Цю можливість відкладено для подальшої розробки, щоб зосередитися на надійності моніторингу та сповіщень.
2.	Інтеграція з національними медичними реєстрами. Для підключення до держреєстрів потрібна додаткова юридична та технічна підготовка, тому реалізацію відкладено.
3.	Підтримка сторонніх фітнес-браслетів та медичних гаджетів. 
Ці рішення дозволяють зосередитися на ключовому функціоналі: надійному зборі та обробці медичних даних, своєчасних сповіщеннях і базовій аналітиці, забезпечуючи стабільність і безпеку першого релізу.
 
4.	BUSINESS CONTEXT (БІЗНЕС-КОНТЕКСТ)
4.3	Operating Enviroment (Робоче середовище)



Робоче середовище для розробки та функціонування програмної системи базується на сучасних технологіях і засобах, що забезпечують надійність,
продуктивність та цілісність системи. У цьому розділі представлено мови
програмування, фреймворки, бази даних та бібліотеки, які будуть використані під час реалізації проєкту.
Мови програмування:
-	Go: використовується для розробки серверної частини. Підтримує роботу розробленого RESTful API та взаємодію із базою даних;
-	JavaScript: використовуватиметься для створення web-застосунку, що надаватиме користувацький інтерфейс для взаємодії із системою.
-	Kotlin: використовується для створення мобільного застосунку, що надаватиме користувацький інтерфейс для взаємодії із системою з мобільного пристрою.
Інфраструктура серверної частини:
-	Gin: веб-фреймворк для створення RESTful API та організації серверної логіки;
-	PostgreSQL: СКБД для зберігання та взаємодією із базою даних користувачів, пацієнтів, отриманих показників тощо;
-	sqlx: бібліотека-розширення до стандартної бібліотеки database/sql, використовується для взамодії із базою даних;
-	JWT: система авторизації на основі токенів, що гарантує безпеку як даних, так і взаємодії із системою загалом.
Інфраструктура фронтенд частини:
-	Vue.js: фреймворк для створення веб-інтерфейсу;
-	Material-UI: бібліотека для полегшення та покращення стилізації веб інтерфейсу;
-	Axious: браузерний http-клієнт для взаємодії із серверною частиною сервісу.
Інфраструктура мобільного застосунку:
-	Jetpack Components та Jetpack Compose для розробки візуальної
складової застосунку;
-	Retrofit – бібліотека для відправки HTTP-запитів до серверної
частини сервісу.
